# Git & GitHub Initialization Guide

This guide summarizes the step-by-step process of initializing a local Git repository, creating essential repository files, and linking it to a remote GitHub account to host a web application.

---

## Part 1: Local Git Configuration

Before running any project commands, Git needs to know who you are so it can associate your name and email with your code snapshots (commits). This configuration is global and only needs to be run once per computer.

```powershell
# Set your name (appears on commits)
git config --global user.name "Your Name or Username"

# Set your email (should match your GitHub email)
git config --global user.email "your.email@example.com"

# Verify your current global settings
git config --global user.name
git config --global user.email
```

---

## Part 2: File Creation (Manual Prep)

Before initializing Git, we create two files manually. These are *not* automatically generated by Git commands, but are industry standard best-practices for any code repository.

| File | Creation Method | Purpose |
| :--- | :--- | :--- |
| **`.gitignore`** | Created Manually | Tells Git which files/folders on your computer should **never** be saved to the cloud (e.g., temporary system files, IDE caches, or node modules). |
| **`README.md`** | Created Manually | A Markdown document describing your project, how to run it, and what technologies it uses. It is displayed on your repository's home page on GitHub. |

---

## Part 3: Local Repository Initialization

Once your folder has files ready, you initialize Git locally in your project directory.

```powershell
# 1. Initialize a new local Git repository
# This command automatically creates a hidden folder named '.git/' in your directory.
git init

# 2. Check which files are new or changed
git status

# 3. Stage your files (prepare them to be saved)
# The '.' means 'add all files in the current directory'
git add .

# 4. Save your staged files (commit the changes)
git commit -m "Initial commit with .gitignore and README"
```

---

## Part 4: GitHub Web Setup

To link your local project to the cloud, you must create a placeholder repository on GitHub:

1. Log into **GitHub.com**.
2. Click the **`+`** icon in the top right -> select **New repository**.
3. Choose a name (e.g., `guess-the-word`).
4. Set it to **Public**.
5. **IMPORTANT:** Do *not* check any boxes for README, `.gitignore`, or license. You want the repository online to be completely blank.
6. Click **Create repository**.

---

## Part 5: Connecting Desktop Git to Web GitHub

Once the GitHub repository is created, link your local folder to it and push (upload) your files:

```powershell
# 1. Add the remote link (names your remote server 'origin')
# Replace USERNAME and REPO-NAME with your actual GitHub details
git remote add origin https://github.com/USERNAME/REPO-NAME.git

# 2. Rename your local branch to 'main' (matching GitHub's default branch name)
git branch -M main

# 3. Push your files to GitHub for the first time
# '-u origin main' tells Git to remember this link for future pushes
git push -u origin main
```

*(Note: The first time you run `git push`, Windows will open a browser tab prompting you to log in and authorize the upload.)*

## File Reference Breakdown

Here is a list of the Git-related folders and files in this project and how they were created:

| Folder / File | Creator | Description |
| :--- | :--- | :--- |
| **`.git/`** | **Automatically created** by `git init` | A hidden directory where Git stores all your snapshots, database, configuration, and history. You should never modify files inside this folder manually. |
| **`.gitignore`** | **Manually created** by developer | A plain text file listing patterns of filenames for Git to ignore. |
| **`README.md`** | **Manually created** by developer | The project landing page file. |
| **`index.html`** / **`style.css`** / **`app.js`** | **Manually created** by developer | Your project code files. Git tracks their history but does not create them. |

---

## Daily Git Workflow (To Save Future Changes)

When you make changes to your code in the future and want to save and push them:

```powershell
# 1. Stage all your changes
git add .

# 2. Commit your changes with a descriptive message
git commit -m "Add new category to words.js"

# 3. Push the changes to GitHub (since origin main is saved, you only need this simple command!)
git push
```
